# 质量评估 Prompt

## 用途

评估生成的图表视觉质量，识别问题并决定是否需要重新生成。

## 图表质量评估 Prompt

### 基础版本

```
你是一位图表可视化质量评估专家。请评估这张图表的视觉质量。

【评估维度】
1. **数据可读性**：数据标签、数值是否清晰可见，是否存在遮挡
2. **布局合理性**：图表元素排列是否合理，是否存在拥挤或空间浪费
3. **标签完整性**：标题、轴标签、图例是否完整且不重叠
4. **颜色对比度**：颜色搭配是否协调，不同数据系列是否易于区分
5. **整体美观性**：图表整体是否专业、清晰、美观

【质量标准】
- **高质量(high)**：图表清晰美观，各元素布局合理，数据易于读取，无明显视觉问题
- **低质量(low)**：存在以下任一问题：
  - 数据标签被遮挡或重叠
  - 图例或标题与图表元素重叠
  - 颜色对比度不足，难以区分数据
  - 布局严重不合理（过于拥挤或空旷）
  - 文字过小无法阅读
  - 坐标轴标签缺失或错误

【输出格式】
请仅输出以下JSON格式，不要添加任何其他文字：
{
    "quality": "high或low",
    "issues": ["问题1", "问题2"]（如果是high质量则为空数组）
}
```

## 参数说明

| 参数 | 类型 | 说明 |
|-----|------|------|
| chart_type | string | 图表类型名称（可选） |

## 输出字段

| 字段 | 类型 | 说明 |
|-----|------|------|
| quality | string | 质量判定：`high` 或 `low` |
| issues | array | 问题列表（high 质量为空数组） |

## 质量判定标准

| 质量等级 | 处理方式 |
|---------|---------|
| high | 继续生成 QA |
| low | 记录问题，可选择重新生成 |

## 常见质量问题

### 数据可读性
- 数据标签被其他元素遮挡
- 数值标注重叠无法辨认
- 数据点过于密集

### 布局合理性
- 图表元素过于拥挤
- 空间浪费严重
- 比例失调

### 标签完整性
- 标题缺失
- 轴标签缺失或被截断
- 图例与图表重叠

### 颜色对比度
- 相邻数据系列颜色相近
- 背景与前景颜色对比不足
- 配色不协调

## 使用方式

1. 将图表图片传递给 VLM 模型
2. 使用上述 prompt 进行评估
3. 解析 JSON 输出
4. 根据 `quality` 字段决定后续流程
